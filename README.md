# Моделирование радиосигнала в городской среде

Интерактивный Python-прототип для моделирования покрытия сотовой связи в городской 3D-сцене. Проект загружает район из OpenStreetMap, строит здания, размещает две базовые станции, трассирует радиолучи с отражениями и показывает горизонтальный срез качества покрытия по SNR.

## Возможности

- построение 3D-сцены района из OSM-файла;
- автоматическое размещение двух вышек на периметре;
- перемещение вышек с сохранением симметрии;
- включение и отключение отдельных вышек;
- движущийся паром как отражающий объект;
- слабый БПЛА как источник внешней помехи;
- сильный БПЛА, временно отключающий ближайшие вышки;
- режимы визуализации: динамические волны, лучи, карта покрытия;
- расчет voxel-карты покрытия и среза по высоте;
- файловый кэш тяжелых расчетов покрытия.

## Структура

```text
.
├── main.py                  # GUI-приложение на PySide6/PyVista
├── city/
│   ├── osm_parser.py        # парсинг OSM и структуры данных
│   ├── projection.py        # перевод lat/lon в локальные метры
│   ├── scene.py             # подготовка сцены и добавление объектов
│   ├── builder.py           # построение PyVista-мешей
│   └── radio.py             # трассировка лучей и расчет покрытия
├── districts/               # OSM-районы для моделирования
├── report/                  # LaTeX-отчет и PDF
└── requirements.txt
```

## Установка

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

Первый запуск может занять несколько секунд: библиотеки визуализации создают локальный кэш в `.cache/`. Дальнейшие запуски обычно быстрее.

## Управление

- слайдер меняет положение вышек на периметре;
- чекбоксы включают и отключают вышки и паром;
- радиокнопки переключают динамические волны, лучи и карту покрытия;
- поля `Лучей`, `Отражений` и `срез` задают параметры расчета;
- кнопка `Слабый БПЛА` включает движущийся источник помехи;
- кнопка `Сильный БПЛА` запускает БПЛА, временно отключающий ближайшие вышки;
- кнопка `Пересчитать` сбрасывает текущий кэш и обновляет покрытие.

## Кэш покрытия

Тяжелые результаты расчета сохраняются в `.cache/coverage`. Это служебная директория, она не хранится в git и может быть удалена. При следующем запуске нужные сценарии будут рассчитаны заново.

## Отчет

Актуальный отчет находится в `report/report.tex`, собранная версия - в `report/report.pdf`.

Пересборка:

```bash
cd report
xelatex report.tex
xelatex report.tex
```
